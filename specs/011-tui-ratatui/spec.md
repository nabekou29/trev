# Feature Specification: TUI 基盤 — イベントループ・ツリー描画・キー入力

**Feature Branch**: `011-tui-ratatui`
**Created**: 2026-02-10
**Status**: Draft
**Input**: User description: "TUI基盤 — ratatui イベントループ、ツリー描画、キー入力"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ツリー表示でディレクトリを閲覧する (Priority: P1)

ユーザーがターミナルで trev を起動すると、指定ディレクトリ（またはカレントディレクトリ）のファイルツリーが画面に表示される。ファイル名・ディレクトリ名がインデント付きで一覧表示され、カーソル位置がハイライトされる。`q` で終了できる。

**Why this priority**: TUI アプリケーションの最も基本的な機能。ツリーが画面に表示されなければ他の操作は全て無意味。

**Independent Test**: trev を起動し、ディレクトリ構造がツリー形式で画面に表示され、`q` で正常終了することを目視確認できる。

**Acceptance Scenarios**:

1. **Given** ファイルを含むディレクトリがある, **When** `trev` を引数なしで起動する, **Then** カレントディレクトリのツリーが画面に表示される
2. **Given** ファイルを含むディレクトリがある, **When** `trev /path/to/dir` で起動する, **Then** 指定ディレクトリのツリーが表示される
3. **Given** ツリーが表示されている, **When** `q` キーを押す, **Then** ターミナルが元の状態に復帰し、trev が正常終了する
4. **Given** ツリーが表示されている, **When** 画面を確認する, **Then** カーソル行が視覚的にハイライトされている
5. **Given** ディレクトリとファイルが混在する, **When** ツリーを確認する, **Then** ディレクトリが先に表示され、ファイルアイコンまたはインジケーターが表示される

---

### User Story 2 - キーボードでツリーをナビゲーションする (Priority: P1)

ユーザーは Vim ライクなキーバインドでツリー内を移動できる。上下移動（`j`/`k`）、ディレクトリの展開・折り畳み（`l`/`h`）、先頭・末尾ジャンプ（`g`/`G`）、半ページ移動（`Ctrl+d`/`Ctrl+u`）が使える。

**Why this priority**: ファイルブラウザとして閲覧以外の操作ができないと実用性がゼロ。表示と並んで P1。

**Independent Test**: trev を起動し、各キーバインドでカーソルが正しく移動し、ディレクトリの展開・折り畳みができることを目視確認。

**Acceptance Scenarios**:

1. **Given** ツリーが表示されている, **When** `j` を押す, **Then** カーソルが 1 行下に移動する
2. **Given** ツリーが表示されている, **When** `k` を押す, **Then** カーソルが 1 行上に移動する
3. **Given** カーソルが折り畳まれたディレクトリ上にある, **When** `l` を押す, **Then** ディレクトリが展開され子ノードが表示される
4. **Given** カーソルが展開されたディレクトリ上にある, **When** `h` を押す, **Then** ディレクトリが折り畳まれ子ノードが非表示になる
5. **Given** カーソルがファイル上にある, **When** `h` を押す, **Then** カーソルが親ディレクトリに移動する
6. **Given** ツリーが表示されている, **When** `g` を押す, **Then** カーソルが先頭に移動する
7. **Given** ツリーが表示されている, **When** `G` (Shift+g) を押す, **Then** カーソルが末尾に移動する
8. **Given** ツリーが表示されている, **When** `Ctrl+d` を押す, **Then** カーソルが半ページ下に移動する
9. **Given** ツリーが表示されている, **When** `Ctrl+u` を押す, **Then** カーソルが半ページ上に移動する

---

### User Story 3 - ディレクトリの遅延読み込み (Priority: P1)

ユーザーがディレクトリを展開すると、子ノードが非同期に読み込まれる。読み込み中は「loading...」等のインジケーターが表示され、UI はブロックされない。読み込み完了後に自動的にツリーが更新される。

**Why this priority**: 大きなプロジェクトでの実用性に不可欠。初回読み込みを待たずにナビゲーション可能であること。

**Independent Test**: 巨大なディレクトリで trev を起動し、展開時にUIがフリーズしないこと、読み込み完了後にツリーが更新されることを確認。

**Acceptance Scenarios**:

1. **Given** 未展開のディレクトリにカーソルがある, **When** `l` で展開する, **Then** UI がブロックされずに操作を続行できる
2. **Given** 子ノードが読み込み中, **When** ツリーを確認する, **Then** 読み込み中であることがわかる表示がされる
3. **Given** 子ノードの読み込みが完了した, **When** ツリーを確認する, **Then** 子ノードが展開されて表示される

---

### User Story 4 - 画面リサイズ対応 (Priority: P2)

ユーザーがターミナルのウィンドウサイズを変更すると、trev の画面が自動的に再描画され、新しいサイズに適応する。

**Why this priority**: 基本的な UX 品質。ただし初期リリースではウィンドウサイズ固定でも最低限動作する。

**Independent Test**: trev 実行中にターミナルウィンドウをリサイズし、表示が崩れないことを目視確認。

**Acceptance Scenarios**:

1. **Given** trev が表示されている, **When** ターミナルを横に広げる, **Then** ツリー表示がリサイズに追従する
2. **Given** trev が表示されている, **When** ターミナルを縦に小さくする, **Then** スクロール位置が適切に調整される
3. **Given** 非常に小さなターミナル (例: 10x5), **When** trev を表示する, **Then** 表示が崩壊せず、可能な範囲で情報が表示される

---

### User Story 5 - ステータスバー表示 (Priority: P2)

画面下部にステータスバーが表示され、現在選択中のファイルのパス、ファイルサイズ、カーソル位置（N/Total）などの情報が確認できる。

**Why this priority**: ユーザーの状況認識を助ける重要な情報だが、ツリー表示・ナビゲーションが先。

**Independent Test**: trev を起動し、カーソル移動に応じてステータスバーの情報が更新されることを目視確認。

**Acceptance Scenarios**:

1. **Given** ファイルにカーソルがある, **When** ステータスバーを確認する, **Then** ファイルの絶対パスが表示されている
2. **Given** ツリーが表示されている, **When** ステータスバーを確認する, **Then** カーソル位置 (例: "3/42") が表示されている
3. **Given** カーソルを別のファイルに移動する, **When** ステータスバーを確認する, **Then** 表示が新しいファイルの情報に更新される

---

### Edge Cases

- ターミナルの幅が非常に狭い場合、ファイル名が途中で切れて表示される
- 空のディレクトリを展開した場合、「(empty)」等の表示がされる
- 読み取り権限のないディレクトリを展開しようとした場合、エラー表示がされてクラッシュしない
- ファイル名に特殊文字（Unicode、絵文字、制御文字）が含まれる場合、表示が崩壊しない
- シンボリックリンクのループがある場合、無限展開にならない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST ターミナルを alternative screen モードで初期化し、終了時に元の状態に復帰する
- **FR-002**: System MUST 指定パス（またはカレントディレクトリ）のファイルツリーを画面に描画する
- **FR-003**: System MUST カーソル位置の行を視覚的に区別して表示する
- **FR-004**: System MUST Vim ライクなキーバインド (`j`/`k`/`h`/`l`/`g`/`G`/`Ctrl+d`/`Ctrl+u`/`q`) を受け付ける
- **FR-005**: System MUST ディレクトリの展開時に非同期で子ノードを読み込み、UI をブロックしない
- **FR-006**: System MUST Nerd Fonts のファイルアイコンをデフォルトで表示し、`--no-icons` フラグでテキストのみ表示に切り替えられる
- **FR-007**: System MUST ツリーのインデントで階層構造を表現する
- **FR-008**: System MUST カーソルが画面外に移動した場合にスクロールして追従する
- **FR-009**: System MUST ターミナルリサイズ時に画面を再描画する
- **FR-010**: System MUST ステータスバーにカーソル位置と選択ファイル情報を表示する
- **FR-011**: System MUST パニックやクラッシュ時にターミナルを復帰させるパニックハンドラを設定する
- **FR-012**: System MUST 読み込み中ディレクトリにローディングインジケーターを表示する

### Key Entities

- **Terminal Session**: TUI アプリケーションのライフサイクル。初期化・描画ループ・クリーンアップ。
- **Event**: キー入力、ターミナルリサイズなどの外部イベント。イベントループで処理される。
- **Viewport**: 画面に表示可能な領域。スクロールオフセットとカーソル位置を管理。

### Assumptions

- キーバインドは Vim スタイル（`h`/`j`/`k`/`l`）をデフォルトとする。将来のカスタマイズは 001-user-config で対応。
- 初回起動時はルートディレクトリの直下 1 階層のみ表示し、サブディレクトリはユーザーの操作で遅延展開する。
- ファイルアイコンは devicons クレートで Nerd Fonts アイコンをデフォルト表示する。Nerd Fonts 未導入環境では `--no-icons` フラグを使用する。
- ステータスバーは画面最下部 1 行に表示する。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが trev を起動してから 1 秒以内にツリーが画面に表示される
- **SC-002**: キー入力からカーソル移動の画面反映まで 16ms (60fps) 以内に完了する
- **SC-003**: 1,000 ファイルを含むディレクトリのツリーで、スムーズにナビゲーション（フレーム落ちなし）できる
- **SC-004**: ディレクトリ展開の非同期読み込み中に他の操作が 100ms 以上ブロックされない
- **SC-005**: `q` キーで終了後、ターミナルが 100% 元の状態に復帰する（代替スクリーン解除、カーソル表示復帰）

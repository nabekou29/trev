## Clarifications

### Session 2026-02-21

- Q: 更新日時の表示フォーマットは？ → A: eza スタイル相対時間（m/h/d/w ago）、1ヶ月以上前は ISO 形式（`2024-01-15`）
- Q: 幅不足時にファイル名を優先してカラムを非表示にするか？ → A: 不要。カラムは常に固定幅で表示し、ファイル名側が切り詰められる。設定でカラムの表示/非表示を選ぶ設計。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ファイルメタデータの表示 (Priority: P1)

ユーザーはツリービューのファイル名の横に、ファイルサイズや更新日時などのメタデータを確認できる。プレビューを開かなくてもファイルの概要を把握でき、ファイル操作の判断が素早くなる。

**Why this priority**: ファイルサイズと更新日時はファイルマネージャーの基本情報であり、カラムシステムの核となる。

**Independent Test**: trev を起動し、ツリービューの各ファイル行にサイズと更新日時が右寄せで表示されていることを確認する。

**Acceptance Scenarios**:

1. **Given** デフォルト設定でtrevを起動, **When** ファイルが含まれるディレクトリを表示, **Then** 各ファイル行の右側にサイズ（人間が読める形式: B, K, M, G）が表示される
2. **Given** デフォルト設定でtrevを起動, **When** ファイルが含まれるディレクトリを表示, **Then** 各ファイル行の右側に更新日時が表示される
3. **Given** ディレクトリノード, **When** ツリーを表示, **Then** ディレクトリのサイズカラムには「-」が表示される
4. **Given** gitリポジトリ内のファイル, **When** git integrationが有効, **Then** gitステータスインジケーターとメタデータカラムが共存して表示される

---

### User Story 2 - カラム表示の設定 (Priority: P2)

ユーザーは設定ファイルで表示するカラムの種類と並び順をカスタマイズできる。狭いターミナル用途（Neovimサイドパネル等）ではカラムを減らす設定にすることで対応する。

**Why this priority**: デフォルトで有用な表示があれば設定なしでも使えるが、ワークフローや画面幅に合わせたカスタマイズ性が必要。

**Independent Test**: config.yml でカラム設定を変更し、trev 再起動後に反映されることを確認する。

**Acceptance Scenarios**:

1. **Given** config.yml で表示カラムをサイズのみに設定（`columns: [size]`）, **When** trevを起動, **Then** サイズカラムのみ表示される
2. **Given** config.yml で表示カラムを空に設定（`columns: []`）, **When** trevを起動, **Then** メタデータカラムは表示されない
3. **Given** config.yml でカラム順序を変更, **When** trevを起動, **Then** 指定した順序でカラムが右側に表示される
4. **Given** 狭いターミナル（30列）でカラム設定がデフォルト, **When** ツリーを表示, **Then** カラムは固定幅で表示され、ファイル名が切り詰められる
5. **Given** config.yml で `modified_at` を `format: absolute` に設定, **When** trevを起動, **Then** 更新日時が常にISO形式（`YYYY-MM-DD`）で表示される

---

### User Story 3 - プレビュー分割比率の設定 (Priority: P3)

ユーザーはプレビューパネルとツリーパネルの分割比率、およびレイアウト切替のブレイクポイントを設定ファイルで変更できる。

**Why this priority**: 現在のハードコードされた比率（50/50, 60/40）とブレイクポイント（80列）で多くのケースをカバーできるため優先度は低いが、カスタマイズできると柔軟性が向上する。

**Independent Test**: config.yml で分割比率とブレイクポイントを変更し、レイアウトに反映されることを確認する。

**Acceptance Scenarios**:

1. **Given** config.yml でツリー幅を60%に設定, **When** 広いターミナルで起動, **Then** ツリー60%・プレビュー40%で表示される
2. **Given** config.yml でブレイクポイントを100に設定, **When** 幅90のターミナルで起動, **Then** 縦分割レイアウトになる
3. **Given** デフォルト設定, **When** 幅80以下のターミナルで起動, **Then** 現行動作を維持（縦分割 60%/40%）

---

### Edge Cases

- ファイル名が非常に長い場合: カラムは常に固定幅で右端に表示され、ファイル名が切り詰められる
- ツリーの深いネスト（10段以上のインデント）: カラムは右端に固定、インデント+名前が残りスペースに収まる
- ファイルサイズ0バイト: 「0B」と表示
- 更新日時が取得できないファイル: 「-」で代替
- シンボリックリンク: 名前の後に「→ ターゲット名」を表示（名前表示の一部として）
- プレビュー非表示時: ツリーが全幅を使い、カラムは正しく表示される
- Nerd Font アイコンの有無: カラム位置に影響しない（右端固定）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ツリービューの各行にメタデータカラム（サイズ、更新日時、gitステータス）を右寄せで表示できること
- **FR-002**: ファイルサイズは人間が読める形式（B, K, M, G）で表示すること
- **FR-003**: ディレクトリのサイズカラムには「-」を表示すること（再帰サイズ計算は行わない）
- **FR-004**: 更新日時は以下のティア形式で表示すること（最大幅10文字）:
  - 1時間未満: `Xm ago`（例: `30m ago`）
  - 24時間未満: `Xh ago`（例: `2h ago`）
  - 7日未満: `Xd ago`（例: `3d ago`）
  - 1ヶ月未満: `Xw ago`（例: `3w ago`）
  - 1ヶ月以上: ISO形式（例: `2024-01-15`）
- **FR-005**: 各カラムは固定の表示幅を持ち、常に右端に表示されること。幅不足時もカラムは非表示にせず、ファイル名側が切り詰められる
- **FR-006**: 設定ファイルで表示するカラムの一覧と順序を指定できること。各カラムは文字列（`- size`）またはオブジェクト形式（`- kind: modified_at` + オプション）で指定可能。`modified_at` カラムは `format` オプション（`relative`（デフォルト）/ `absolute`）を持つ
- **FR-007**: プレビューパネルとツリーパネルの分割比率を設定ファイルで指定できること
- **FR-008**: 横分割/縦分割の切替ブレイクポイント（ターミナル幅）を設定ファイルで指定できること
- **FR-009**: 既存のgitステータスインジケーターはカラムシステムの一部として統合すること
- **FR-010**: シンボリックリンクノードはファイル名の後に「→ ターゲット名」を表示すること（名前表示の一部として、独立カラムにはしない）

### Assumptions

- ファイルサイズと更新日時はOSメタデータから取得済み（TreeNode に `size: u64` と `modified: Option<SystemTime>` が存在）
- パーミッション表示は将来拡張とし初期スコープに含めない（TreeNode にパーミッション情報がないため）
- ディレクトリの再帰サイズ計算はパフォーマンス影響が大きいため行わない
- シンボリックリンクのターゲットパス取得は新たに必要（現在 `is_symlink` フラグのみで、ターゲットパスは未保持）

### Key Entities

- **Column**: 表示可能な情報カラム。種別（size, modified_at, git_status）と固定表示幅を持つ。種別ごとにオプション設定を持てる
- **ColumnLayout**: 設定されたカラム一覧に基づいて各行の右端領域を構成するルール
- **LayoutConfig**: プレビュー分割比率とブレイクポイントを含むレイアウト設定

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: デフォルト設定でファイルサイズと更新日時がツリービューに表示される
- **SC-002**: ターミナルリサイズ時にカラムレイアウトが即座に再計算される
- **SC-003**: Neovimサイドパネル（幅30列）でもカラムが正しく表示され、操作可能である
- **SC-004**: 1000ファイルのディレクトリでもカラム表示追加によるスクロール性能の劣化が体感できない
- **SC-005**: 設定ファイルの変更がアプリ起動時に正しく反映される
